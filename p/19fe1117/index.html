发表于 2019-04-20 | 分类于 docker | | 热度 ℃
字数: 1.1k | 时长 ≈ 5

背景

使用容器目的就是为了实现自动化、服务编排、遥测数据、高可用等更易于管理的基础设施，所以必须由一个管理设施来控制容器的生命周期、管理服务集群的网络、存储。
在容器管理平台界有三剑客，分别为swarm、mesos、k8s，其中k8s功能最为强大，夺得卫冕之王，所以我这里主要学习和记录docker+k8s的搭建和使用过程。

概念性的东西必须要了解，可以到

  • k8s官方文档
  • k8s手册
    逐章看一下概念，了解k8s的kubelet、kube-proxy、master/node、pod、controller、namespace、权限、网络、存储等概念再进行实操。

其中CRI、CNI、CSI分别为开放容器接口、开放网络接口和开放存储接口，可以通过调用API自己实现对应插件，我这里分别使用docker、flannel、(?)实现，少走弯路。
